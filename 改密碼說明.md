# 後台密碼修改說明

這份說明用來修改 `wowma.onrender.com` 的後台登入密碼。

後台登入網址：

```text
https://wowma.onrender.com/admin.html
```

## 一、先準備新密碼

建議密碼至少 12 個字以上，混合英文大小寫、數字，例如：

```text
Wowma2026Store!9
```

不要使用：

```text
admin123
password
生日
電話
```

## 二、把新密碼轉成雜湊值

在 Windows PowerShell 貼上下面這段，按 Enter：

```powershell
$newPassword = Read-Host "輸入新後台密碼"
$sha = [System.Security.Cryptography.SHA256]::Create()
$bytes = [System.Text.Encoding]::UTF8.GetBytes($newPassword)
$hash = $sha.ComputeHash($bytes)
[System.BitConverter]::ToString($hash).Replace("-", "").ToLower()
```

PowerShell 會要求你輸入新密碼，輸入後會顯示一長串英文數字。

範例長得像這樣：

```text
e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855
```

請複製這一整串，等一下要貼到 Render。

## 三、到 Render 修改環境變數

1. 打開 Render Dashboard。
2. 進入你的服務 `wowma`。
3. 左側點 `Environment`。
4. 找到或新增這個環境變數：

```text
ADMIN_PASSWORD_HASH
```

5. 把剛剛產生的那一長串雜湊值貼到 Value。
6. 如果有看到 `ADMIN_PASSWORD`，建議刪除，或不要使用它。
7. 按 `Save Changes`。

## 四、重新部署

Render 儲存環境變數後通常會自動重新部署。

如果沒有自動部署：

1. 進入 `wowma` 服務。
2. 點右上角或頁面上的 `Manual Deploy`。
3. 選 `Deploy latest commit`。
4. 等部署狀態變成 `Live`。

## 五、測試新密碼

1. 打開：

```text
https://wowma.onrender.com/admin.html
```

2. 帳號輸入：

```text
admin
```

3. 密碼輸入你剛剛設定的新密碼。
4. 確認可以登入。
5. 再試舊密碼，確認舊密碼不能登入。

## 注意事項

- Render 後台看到的是密碼雜湊值，不是原始密碼。
- 忘記密碼時不用找回，只要照這份說明重新產生一組新的 `ADMIN_PASSWORD_HASH`。
- 改完密碼後，正在登入的舊工作階段可能暫時還會有效；如果要保險，可以重新部署一次，或關掉瀏覽器再重新登入測試。
